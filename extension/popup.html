      How Tabduct works

      What it is

      Tabduct connects an AI coding agent (such as Claude Code) to the real Chrome you are already using. Instead of driving a fresh, empty browser, the agent works with the exact tabs you hand it — your sessions, your logins, your open pages — so it can help with things that normally need a signed-in browser.

      Quick start

      1. Start — click the button (or the header dot); it turns green.
      2. Point your agent at it — open Settings (⚙), copy the MCP endpoint + Authorization token into your agent's MCP config (template below), reload the agent.
      3. Share — Share Current Tab for one tab, or Share Everything. The agent can now work with what you shared, and nothing else.

      What the agent can do

      For a tab you have shared, the agent can read the page's text and a compact outline of its elements, take a screenshot, click, type, wait for things to load, read the console, navigate, run in-page scripts, and open/close tabs. Most of these work even on strict-CSP sites (GitHub, banks) — see below. Everything you have not shared stays completely invisible — it can't even see the title of an unshared tab. Several safety limits below can narrow this further.

      Sharing & how you stay in control

      • Two ways to share: Share Current Tab (just the tab you're on) or Share Everything (all current and future tabs). Sharing resets when you restart Chrome, so you never leave something shared by accident.
      • Origin filter (Settings) decides which sites are eligible at all, and overrides every sharing mode:
        • Block mode (default): the listed sites are never shared — put your email, bank, etc. here.
        • Allow mode: flips it — only the listed sites can be shared, everything else is blocked (an empty allow list blocks everything). Best if you want the agent confined to a known set of sites.
      • Lock shared tabs to their domain (Settings, on by default): a shared tab that navigates to another site loses access — so a shared shopping tab can't quietly follow you into your bank. Turn it off to let the agent browse freely within a shared tab.
      • Don't auto-share tabs the agent opens (Settings, on by default): tabs the agent opens itself aren't shared until you approve them, so it can't silently reach a brand-new site.
      • Read-only (Settings): the agent may look (read text, screenshot) but never click, type, navigate, run scripts, or open/close tabs.
      • Auto-expire (Settings): un-shares everything after a time you pick (5 min … 10 h).

      Marking shared tabs & indicators

      • Header dot: green = running, red = stopped. Click it to start / stop.
      • Toolbar icon: plain = idle; green dot = the current tab is shared; red dot = Tabduct is stopped.
      • "⚡" tab group (Settings, on by default): shared tabs are gathered into a coloured ⚡ group. It's two-way — drag a tab into the group to share it, drag it out to unshare.
      • Right-click a page → "⚡ Tabduct: share / unshare this tab" — the quickest way to toggle one tab.
      • Shortcut Ctrl+Shift+Y toggles sharing for the current tab.

      Strict-CSP sites & CDP mode (advanced)

      Clicking, typing, reading, and waiting work everywhere — they run as injected functions, which a site's Content-Security-Policy can't block. Only arbitrary JavaScript (execute_script) can be blocked by a strict CSP (GitHub, banks, most SaaS). For that, the Advanced settings offer opt-in CDP mode: eval bypasses CSP via the DevTools Protocol, with a developer mode that routes all eval through it and captures the full console (including uncaught errors and network/CSP warnings). It's OFF by default — nothing attaches until you flip the toggle. (Chrome does not allow requesting "debugger" at runtime, so it's a required permission granted at install; the toggle is what actually authorizes attaching.) While on, Chrome shows a "this tab is being debugged" banner (your signal), a "CDP" chip appears in the header, and it still obeys consent — never in read-only.

      Starting Tabduct

      Click Start (or the header dot). Tabduct launches a small server on your own machine — nothing is exposed to the internet — and shows an endpoint URL and a secret token in Settings. Point your agent at those. Click the dot again to Stop it (the server process exits).

      Shared hub is on by default: Tabduct exposes one fixed endpoint (127.0.0.1:12311) with a stable token, so your agent config never changes — and every browser you connect appears behind that single endpoint. You can turn the hub off in Settings if you'd rather point the agent at a direct, per-browser endpoint.

      Configuring your agent (MCP)

      Tabduct speaks MCP (Model Context Protocol) over HTTP. After connecting, copy the exact endpoint and token from Settings, then add a server entry to your MCP client's config:

      {
  "mcpServers": {
    "tabduct": {
      "type": "http",
      "url": "http://127.0.0.1:12311/mcp",
      "headers": {
        "Authorization": "Bearer PASTE_TOKEN_FROM_SETTINGS"
      }
    }
  }
}

      Replace the port and token with the values shown in Settings once running (the shared hub uses port 12311; a single browser uses whatever port it reports). Reload your agent and it will discover the Tabduct tools: list_tabs, get_active_tab, get_page_content, get_dom_snapshot, screenshot, wait_for, click, type, navigate, open_tab, close_tab, activate_tab, get_console_logs, list_network_requests, get_network_request and execute_script.

      Running several browsers or profiles

      Install Tabduct in each Chrome profile you want to use — each Google account / profile is separate, so share and start in each one. With the hub on, they all appear behind the one endpoint and the agent tells them apart by their Label (auto-named like Chrome-abcd). Give each a clear name in Settings — e.g. Work and Personal — so the agent (and you) never confuse them.

      Good to know

      • The server listens only on 127.0.0.1 (your machine) and is guarded by the token — treat the token like a password and don't paste it into untrusted places.
      • The origin filter is the only thing that confines which sites the agent can reach. In Block mode with an empty list, an agent that can open tabs could reach any non-listed site — use Allow mode (or add sensitive sites to the block list) if that matters to you.
      • If the agent asks for a tab that isn't shared, is blocked, or has expired, it simply gets "not shared" and moves on — no error, no leak.
